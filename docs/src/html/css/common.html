路由页面公共样式
css.conf中的common属性用于定义部分路由页面的公有样式。它的值是一个数组，数组中的元素是一个路由页面组，他们共享一套样式。路由页面组使用router属性定义，值是一个数组，样式使用css属性定义，这一套样式可以使定义在配置文件中的字符串，也可以是 /src/css/ 目录中的css文件， 可以是一个或者多个css文件，多个文件使用逗号隔开，请看几个例子。
1.使用css字符串
使用router属性定义路由页面组，为了简单起见，这里只添加当前页面，也就是 /css/common 路由页面，读者可以自行添加多个。
css.conf 文件：
{ variable:{ }, common:[ { router:["/css/common"], css:".test-common1{background-color:#aaa}" },{ router:["/css/common"], css:".test-common2{color:#f44}" } ] }
使用：
<div class='test-common1'>样例1</div> <div class='test-common2'>样例2</div>
以下是输出的结果：
样例1
样例2
2.使用css文件名
css属性中可以使css文件名，css文件存放在 /src/css/ 目录下。
test1.css 文件：
.test-common3{background-color:#00a}
test2.css 文件：
.test-common4{color:#88f}
test3.css 文件：
.test-common5{color:#f5f}
css.conf 文件：
{ variable:{ }, common:[ { router:["/css/common"], css:"test1.css" },{ router:["/css/common"], css:"test2.css,test3.css" } ] }
使用：
<div class='test-common3'>样例3</div> <div class='test-common4'>样例4</div> <div class='test-common5'>样例5</div>
以下是输出的结果：
样例3
样例4
样例5
