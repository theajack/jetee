use()
使用 Jet.router.use() 方法启用路由模块，并且设置路由规则，参数是一个json。请看一个例子：
Jet.router.use({ history:false, base:"/jet", trueBase:false, index:'/', router:{ '/index':'/index', '/intro':{ name:'/intro/index?id=2', children:{ '/install':'/intro/install', '/contents':'/intro/contents', '/html':'/intro/html', '/grammer':'/intro/grammer', } }, '/attr':{ name:'/attr/index', params:{ id:1 } } } })
history参数 表示是否启用history模式，关闭history模式则在链接中会含有 # ，但是对后端开发人员要求较小。 若是开启了history模式，则链接中不含有 #，但是需要后端开发人员将所有访问地址都指向index.html文件。该参数默认为false。
base参数 表示是否在host和路由地址之间增加一级地址，改地址可以是虚拟的也可以是真实的文件目录。
trueBase参数 用于指明base参数设置的增加的根目录地址是否是真实的。该参数默认为false，在没有设置base参数是，该参数不起作用。
index参数 用于指定应用程序的入口路由地址。
map参数 是最重要的一个参数，它是一个json数据，将路由地址映射成真实的html文件。 json的键是一个路由地址，值是字符串类型，表示在 /src/html 文件夹中的一个文件，可以使用多重文件夹嵌套，另外.html的后缀可以省略，如果值与键相等，则可以使用空字符串代替值。 如果使用了资源管理，则也可以使用 资源管理的方式表示html文件地址。
对于get类型请求的参数，如上面的例子，可以在值后面带上 ?parame=v... 。 值除了可以使字符串还可以是json数据，json数据中可用的属性有 name、params和children，name指的是 路由地址html文件路径， params指的是get参数，是一个json，params和直接写在文件地址后面只能使用一种；children指的该地址下的路由地址。
