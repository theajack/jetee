Jet官方库
Jet目前官方提供了以下几个库供开发者使用：
1.render-time (用于打印项目依赖加载时间、页面初始化和渲染时间等信息的库。)
2.router (官方路由库)
3.css-config (css语法和配置文件库)
4.res (Jet的统一资源管理库)
5.valid (输入验证和表单验证)
6.lang (国际化语言库)
7.module (Jet自由的模块系统)
8.tool (工具方法，JUI的依赖项)
9.jui (完美契合Jet的UI框架)
10.babel (使用es6进行开发)
11.less (使用less进行开发)
若是您使用的是 npm安装方式或手动下载Jet项目 ，也就是使用完整的Jet架构，则只需使用 下文中提到的 use方法加载这些依赖。 但是如果您是使用标签引入的方式，也就是没有使用完整的Jet架构，那么您如果需要使用这些官方库的话，需要手动使用script方式引入，引入方式如下:
<!--{name}指的就是库的名称--> <script src='https://www.theajack.com/jet/assets/js/jet-lib/{name}.js'> <script/>
建议下载到本地之后再引用。
若是要使用jui，虽然Jet会自动加载jui相关的css，但是处于渲染效果和性能考虑，推荐在head标签中引入 jui.css文件，如下.
<!-- JUI css --> <link rel="stylesheet" href='https://www.theajack.com/jet/assets/css/jui.min.css'/>
我们将在下面的章节中详细介绍这些库的功能和api.
第三方库
开发者可以使用任何其他第三方库。
与官方库一样， 若是您使用的完整的Jet架构开发，则只需使用 下文中提到的 use方法加载这些第三方库。 如果不是则，需要手动使用script方式引入，引入方式如下:
<!--{src}指的就是第三方库的url，可以是本地文件或是http链接--> <script src='{src}'> </script>
对于第三库也是建议下载到本地之后再引用。
Jet.canUse
使用 Jet.canUse() 方法可以判断库是否可用，该方法接受一个字符串类型参数，返回一个bool类型的值。
